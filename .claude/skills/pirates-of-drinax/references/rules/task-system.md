## The check (2D6 mechanic) (Core p.58)
- A **check** = roll **2D6** + all applicable Dice Modifiers (DM), compare total to a target number. Total ≥ target = **success**; below = **failure**.
- Default (Average) target is **8+**. If no difficulty is stated, assume Average (8+). (Core p.60)
- **Characteristic check:** 2D + characteristic DM (no skill). Used when no skill applies (e.g., raw STR to force a door, DEX to balance).
- **Skill check:** 2D + skill level + characteristic DM + any other DMs ≥ target. Referee picks which characteristic fits the situation. (Core p.59)
- The skill+characteristic pairing is set by the referee per use, not fixed. Examples: shoot = Gun Combat + DEX; spot gun type from a wound = Gun Combat (or Investigate) + INT; hide = Stealth + DEX; stay still to avoid detection = Stealth + END; carouse generally = Carouse alone; out-drink someone = Carouse + END.
- INT vs EDU overlap: INT = inspiration, improvisation, pure reasoning, unfamiliar problems; EDU = trained responses, known solutions, recalled facts. No EDU helps a totally unfamiliar problem; no INT recalls trivia not learned beforehand. (Core p.59)

## When to call for a check (Core p.58)
Most actions need **no** check (Skill 0 lets you fly an air/raft, run, pull library data with no roll). Call for a check only when:
1. The Travellers are in danger.
2. The task is especially difficult or hazardous.
3. The Travellers are under time pressure.
4. Success/failure is especially important or interesting.

**Automatic success:** If the crew have the requisite skill and failure carries no consequence, just rule success — do not make them roll until they pass (Core p.60). Call for the check only when failure has a real consequence (e.g., a failed Streetwise picks up a false rumour).

## Dice Modifiers (DM)

### Characteristic DM by score (Core p.58)
| Characteristic score | DM |
| --- | --- |
| 0 | −3 |
| 1–2 | −2 |
| 3–5 | −1 |
| 6–8 | +0 |
| 9–11 | +1 |
| 12–14 | +2 |
| 15+ | +3 |

### Skill levels (Core p.58)
- **No level (untrained):** suffer **DM−3** on the check for being unskilled.
- **Skill 0:** competent but inexperienced — grants **no bonus** but **avoids the −3 untrained penalty**.
- **Skill 1+:** trained — **+1 DM per level** to checks using that skill (Skill 1 = +1, Skill 2 = +2, etc.). Levels 2–3 = skilled professional; 4–5 = renowned.

### Stacking
- Total = 2D + skill level + characteristic DM + any other listed DMs (gear, range, cover, situational modifiers from the rules). All applicable DMs add together.
- Worked example: Erik (INT 9 → +1, Carouse 1 → +1) on a Difficult (10+) Carouse check rolls **2D+2** vs 10. (Core p.60)
- Worked example (untrained): Morn (DEX 12 → +2, no Stealth → −3) rolls **2D−1** vs 8. (Core p.59)

> Note: ordinary numeric **DMs** are "hard-wired" — apply only when a rulebook/supplement lists them. Situational advantage/disadvantage invented on the fly is handled by **Boon/Bane**, not by ad-hoc DMs. (Core p.64)

## Task Difficulty table (Core p.60)
| Difficulty | Target | Typical use |
| --- | --- | --- |
| Simple | 2+ | Trivial for anyone (often not even rolled): request weather data, order a meal, sneak past a drugged guard. |
| Easy | 4+ | Trivial for a pro, simple for an amateur: hike cross-country, hold level flight, use a simple interface. |
| Routine | 6+ | Easy even for amateurs: hit a prone foe in melee, land in optimum conditions w/ computer aid, translate a newspaper. |
| Average | 8+ | Moderate obstacle for a pro: accurate ranged shot, plot jump, repair a circuit board, find a buyer. |
| Difficult | 10+ | Tough even for a pro: shoot in a storm, defend a guilty client, hack a secure network. |
| Very Difficult | 12+ | Hard for a pro, near-impossible for amateurs. |
| Formidable | 14+ | Exceptionally hard: surgery on alien biology, hack military-grade network, disarm a bomb. |
| Impossible | 16+ | Near-miracle: reconstruct alien DNA, rebuild a worldwide military network from scratch. |

## Effect (margin of success/failure) (Core p.61)
- **Effect = total rolled (incl. all DMs) − target number.** Positive = degree of success; negative = degree of failure.
- Example: roll 4+3=7, +DM3 = 10, on Average (8+) → Effect = 10 − 8 = **2**.
- Effect is also used to calculate combat damage and to feed Task Chain DMs (below).

### Effect Results table (Core p.61)
| Effect | Result |
| --- | --- |
| −6 or less | **Exceptional Failure** — fails as completely as possible; anything that can go wrong does (device further damaged/destroyed; deeper social trouble). |
| −2 to −5 | **Average Failure** — task failed. |
| −1 | **Marginal Failure** — almost made it; referee MAY allow a scraped success at a significant cost (drop the backpack, ship never flies again, hack succeeds but you're traced). |
| 0 | **Marginal Success** — barely succeeds, often with a condition (engine fixed but overheats; buyer found but untrustworthy). |
| 1 to 5 | **Average Success** — succeeds normally. |
| 6 or more | **Exceptional Success** — succeeds impressively/elegantly. |

## Boon & Bane (Core p.60–61)
- **Boon** (help: good tools, competent aid, favourable circumstances): roll **3D6, keep the highest 2**.
- **Bane** (hindrance: poor tools, dim light, bad circumstances): roll **3D6, keep the lowest 2**.
- Only **one** Boon or Bane can apply to any single check. Multiple Boons (or multiple Banes) still count as just one. A Boon and a Bane on the same check **cancel out** (roll straight 2D).
- Boons/Banes are referee fiat for on-the-fly advantage/disadvantage and never appear pre-printed in rulebooks/adventures. (Core p.64)

## Opposed checks (Core p.62)
- Two characters directly opposing each other: both make their checks; **highest Effect wins**.
- **Tie (draw):** standstill, neither gains advantage; they may try again or events may move on.

## Multiple tasks at once (Core p.62)
- Doing two or more things simultaneously (e.g., fly + fire weapons): for **every** extra task, raise the difficulty of **each** task by **one level** (Easy→Routine, Very Difficult→Formidable, etc.).

## Timeframes & going faster/slower (Core p.62)
- Each task takes **1D × (its time unit)**. Unit depends on the task.

| Timeframe | Increment | Example action |
| --- | --- | --- |
| 1D seconds | 1 second | Shooting, punching, jumping |
| 1D combat rounds | 6 seconds | Hurrying a maths problem |
| 1D × 10 seconds | 10 seconds | Rerouting power, opening a comms channel |
| 1D minutes | 1 minute | Applying first aid, basic technical tasks |
| 1D × 10 minutes | 10 minutes | Complex technical tasks, searching an area thoroughly |
| 1D hours | 1 hour | Building a shelter, moving through wilderness |
| 1D × 4 hours | 4 hours | Researching a problem |
| 1D × 10 hours | 10 hours | Repairing a damaged ship |
| 1D days | 1 day | Combing a city for a missing person |

### Going faster or slower (Core p.63)
- Before rolling, you may shift **one** step on the Timeframes table:
  - **Faster** (shorter increment): **DM−2** to the check.
  - **Slower** (longer increment): **DM+2** to the check.
- Only one step in either direction.

## Task chains (linked checks) (Core p.63)
- A **task chain** = a sequence where one check's result feeds a DM into the next (e.g., Streetwise to find a fence → Broker to negotiate; or Recon → Tactics → Stealth/Electronics for an assault).
- After each check, apply its Effect as a DM to the **next** check via the Task Chain table:

| Previous check | DM to next check |
| --- | --- |
| Failed, Effect −6 or less | −3 |
| Failed, Effect −2 to −5 | −2 |
| Failed, Effect −1 | −1 |
| Succeeded, Effect 0 | +1 |
| Succeeded, Effect 1 to 5 | +2 |
| Succeeded, Effect 6 or more | +3 |

- One Traveller doing multiple checks on one job can chain them if the referee agrees each affects the next.

### Working together / cooperation (Core p.63)
- When several Travellers work the **same** task at the same time: one makes the **final** check; the others make checks first and feed their Effects in as Task Chain DMs to that final check.
- Example: Erik forces a door (STR check); Kathya makes an STR check and applies her Effect as a Task Chain DM to Erik's check.

## Order of operations (Core p.64)
1. **Difficulty:** referee sets it from how hard the task is *without* outside factors (a phone call stays Simple even mid-storm).
2. **Dice Modifiers:** apply only DMs the rules list (skill, characteristic, gear, etc.).
3. **Boon/Bane:** add one Boon or Bane only if an external factor helps or hinders.

In most cases: set a Difficulty, decide Boon/Bane if any; the rules supply any needed DMs.

## Quick procedure
1. Decide if a check is even needed (danger/difficulty/time/stakes); else rule auto-success.
2. Set **Difficulty** → target number.
3. Pick **skill + characteristic** (or characteristic alone); total **2D + skill + char DM + other DMs**. Untrained = −3.
4. Apply **one Boon or Bane** if circumstances warrant (3D keep best/worst 2).
5. Optionally shift timeframe ±1 step (faster −2 / slower +2).
6. Roll. Total ≥ target = success.
7. Compute **Effect** = total − target; read Effect Results; if chaining, carry Effect → next check's DM.
